# **プロジェクト：論文スワイプ・レコメンダー（PaperReel）**

## **1\. プロジェクト概要**

研究者や大学生、大学院生などのアカデミアに向けた、スマホ向け論文探索アプリケーション。YouTube ショートやTikTokのような「パパッと流せる」UX を通じて、ユーザーの潜在的な興味を学習し、未知の論文との出会い（セレンディピティ）を提供する。論文の推薦にはレコメンドエンジンを活用し、ユーザの興味にあった論文、研究分野に関連する論文を提示するほか、これまでまったく関わってこなかった分野の論文（しかし、その分野の視点やアプローチがヒントになったり単純に知的好奇心を刺激したりする）ような設計になっている。

## **2\. 推奨エンジンの設計指針**

YouTube のレコメンドアルゴリズム（Covington et al., 2016）をベースとした 2 段階構成を採用する。（該当の論文PDFはディレクトリ内に45530.pdfとして配置されている）

### **A. 2 段階アーキテクチャ**

1. **Candidate Generation（候補生成）**:  
   * 数百万の論文から 100〜数百件を抽出。  
   * **初期（コールドスタート）**: ユーザーがオンボーディングで入力した「研究分野」「年齢」等のプロフィール情報、および検索クエリやその他取得した諸データをもとに、LLM を用いたタイトル/アブストラクトのベクトル化による内容ベースフィルタリングを行う。  
   * **運用期**: ユーザーの過去の行動（Good/Bad）に基づく協調フィルタリングを統合。  
2. **Ranking（ランキング）**:  
   * 抽出された候補に精密なスコアを付与。  
   * 特徴量：特定学会への好み、論文の鮮度（Example Age）、スワイプ面での滞在時間予測など。  
   * ランキング方式に関しては今後改善していく（今回はデモのためダミーを用いる）。

### **B. 学習信号（ログ収集）**

* **明示的信号**: スワイプによる Good / Bad 評価。  
* **暗黙的信号**:  
  * **滞在時間 (Dwell Time)**: スライドを閲覧していた秒数。  
  * **詳細遷移**: Abstract の展開や DOI リンクへのクリック（強い関心とみなす）。  
  * **ホームでのクリック**: どのサムネイル（ショートフィード）に惹かれたか。

## **3\. UI/UX 仕様**

### **A. インターフェース**

* **スマホ専用設計**: 縦型の全画面カード UI（アスペクト比 9:16 を意識）。  
* **論文サマリー表示**: 以下の 4 項目で構成される「1 枚まとめスライド」形式。  
  * **背景 (Background)**: 研究の動機（オレンジ系の配色）。  
  * **目的 (Objective)**: 解決したい課題（青/紫系の配色）。  
  * **提案 (Proposal)**: 提案手法やシステム（緑系の配色）。  
  * **評価 (Evaluation)**: 実験結果や成果（青系の配色）。  
* **詳細表示**: Abstract をアコーディオン形式で展開可能。

### **B. インタラクション**

* **右スワイプ (Good)**: ポジティブ評価としてログを記録し、次のカードへ。Goodした論分はGoodした論文としてのリストに追加される。  
* **左スワイプ (Bad)**: ネガティブ評価としてログを記録し、次のカードへ。  
* **上スワイプ (Undo)**: 1 つ前の論文に戻る。直前のログを削除または「取り消し」としてマークする。  
* **クリック/タップ**: Abstract の展開。  
* **ホーム遷移**: 画面下部のナビゲーション、またはスワイプ画面からのバック操作。

## **4\. ユーザー体験（UX）の詳細フロー**

1. **オンボーディング**:  
   * 初回起動時、研究分野、年齢、興味のある技術スタック等の諸条件を回答。（マッチングアプリの初回をイメージ）  
   * このデータが Candidate Generation の初期シードとなる。プロフィール登録完了後、ホーム画面へ遷移。  
2. **ホーム画面（YouTube 形式）**:  
   * 上部に検索バー、メインエリアには「ショートフィード」が 2 列・複数段のグリッド形式で表示される（参考までにYoutubeの画面をYoutube.PNG として置いておくので参考にしてください）。  
   * 各フィードはスワイプモード（Reel）への入り口となるサムネイル。  
3. **ショートフィード（スワイプ/Reel）**:  
   * ホームのフィードをタップ、または「探索」から全画面モードへ。  
   * ランキング 1 位の論文カードから順に表示され、ユーザーが閲覧を開始した瞬間から dwell\_time の計測を開始。  
4. **評価アクションとアニメーション**:  
   * スワイプ完了後、即座に次の論文（2 位、3 位...）が下または横からスライドしてくる。  
   * 100 枚のスワイプ完了時、または区切りで「ログ確認」や「再探索」の導線を提示。  
5. **Undo（修正）**:  
   * 上にスワイプすることで即座に直前のカードを画面内に引き戻す。Slack のキャッチアップ機能のようなスムーズな挙動。  
6. **ライブラリ（後で読む）**:  
   * 「Good」した論文は、ユーザープロフィールの「Goodした論文」セクションからいつでも再確認可能。

## **5\. 実装要件（デモ段階）**

### **A. モジュール化設計**

「UI レイヤー」と「エンジンレイヤー」をインターフェースで分離。

* **入力**: ユーザープロフィール（分野、年齢）、検索キーワード、過去のアクションログ。  
* **出力**: ランキング済み論文データ（100件）のリスト。

### **B. 技術スタック（推奨）**

* **Frontend**: React / Next.js  
* **Styling**: Tailwind CSS  
* **Icons**: Lucide-React  
* **Animation**: Framer Motion（スワイプおよびグリッドと全画面の遷移アニメーションのため）

### **C. データ構造（想定）**

{  
  "id": "論文ID",  
  "title": "タイトル",  
  "authors": "著者リスト",  
  "content": {  
    "background": "テキスト",  
    "objective": "テキスト",  
    "proposal": "テキスト",  
    "evaluation": "テキスト"  
  },  
  "abstract": "全文",  
  "score": 0.95,  
  "metadata": {  
    "field": "研究分野",  
    "venue": "学会名",  
    "year": 2024  
  }  
}
